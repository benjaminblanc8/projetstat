\chapter{Objectif de ce document}\label{objectif-de-ce-document}

Ce document sert de modèle pour les rapports de projets de première
année à l'Ensai avec quelques recommandations et consignes de base pour
la forme. Ce modèle ne traite pas du fond et, en aucun cas, ce document
ne se substitue aux consignes données par chaque responsable de projet
qui seront toujours prépondérantes par rapport aux éléments figurant
ici.

Une fois ce document vidé par les étudiants de son contenu explicatif,
il en restera le « squelette » qui pourra servir comme architecture de
départ à la rédaction de tout rapport.

Le contenu explicatif est dans le PDF mais aussi dans les commentaires
figurant directement dans le document Quarto sur RStudio. De manière
générale, on peut mettre des commentaires dans ce type de document de
deux façons différentes selon l'endroit :

\begin{itemize}
\tightlist
\item
  si dans une portion de code R, derrière un croisillon (« \texttt{\#}
  »)\footnote{Parfois appelé par erreur dièse mais le dièse a le symbole
    suivant « ♯ » différent de « \# » ; parfois appelé aussi
    \emph{hashtag} en français mais \emph{hash} est un des noms du
    symbole en anglais et le \emph{hashtag} désigne en anglais un
    mot-clef c'est-à-dire un mot collé tout de suite après un croisillon
    et servant de mot référence.} et une espace\footnote{En typographie,
    le mot « espace » est féminin.} ;
\end{itemize}

\begin{itemize}
\tightlist
\item
  si dans le texte hors portion R, entre « \texttt{\textless{}!-\/-} »
  et « \texttt{-\/-\textgreater{}} ».
\end{itemize}

\chapter{Précisions sur la table des
matières}\label{pruxe9cisions-sur-la-table-des-matiuxe8res}

Pour afficher la page deux rangs avant celle-ci, il faut saisir la ligne
« \texttt{\textbackslash{}tableofcontents} » à l'endroit souhaité. Par
défaut, la page est appelée « Table des matières », si on veut la
renommer « Sommaire », on fait précéder la ligne précédente par la ligne
«
\texttt{\textbackslash{}renewcommand\{\textbackslash{}contentsname\}\{Sommaire\}}
».

→ Quelle différence entre un sommaire et une table des matières ?

Généralement, un sommaire est plus ou moins détaillé (parfois juste avec
les grandes parties et sans numéro de page) et toujours en début de
document quand une table des matières, plus souvent en fin de document,
rentre dans les détails de chaque section toujours liée à la page où
elle débute. Dans les faits, les deux termes sont souvent employés l'un
pour l'autre, surtout quand c'est détaillé (ce qui est plus habituel
d'une table des matières) tout en étant à la fois en début de document
(ce qui est plus habituel d'un sommaire). Étant ce qui est attendu ici,
le nom de cette page sera à la convenance personnelle des rédacteurs du
rapport.

Dans le PDF généré en ligne, les titres sont cliquables et renvoient aux
sections concernées plus bas dans le document.

Attention : contrairement aux logiciels de traitement de texte les plus
couramment utilisés (comme Microsoft Word ou Libre Office Writer), la
page de titre n'est pas prise en compte pour la numérotation des pages
(un peu comme s'il s'agissait de la couverture d'un livre). Pour ça,
pour ceux qui connaissent, c'est pareil que pour un document tapé avec
LaTeX sur Overleaf. La page 1 est la première page après la page de
titre soit ici celle de la table des matières.

\chapter{Utilité de Quarto et
utilisation}\label{utilituxe9-de-quarto-et-utilisation}

Un document Quarto sur RStudio permet de mêler du texte et du code R
exécutable pour tout regrouper dans un seul document finalisé, comme ici
un rapport en format PDF. Pour en savoir plus sur Quarto :
\href{https://quarto.org}{site officiel de Quarto}\footnote{Pour faire
  un lien cliquable vers une page externe, voir dans
  \hyperref[lienexterne]{la section sur les liens externes}.} (en
anglais).

L'avantage par rapport aux autres logiciels de traitement de texte est
que cela permet d'intégrer du code directement dans le document et
d'afficher les résultats (graphiques, tableaux, etc.) automatiquement.
Cela favorise une reproductibilité scientifique : on peut voir et
exécuter le code à l'origine des résultats.

Les résultats sont ainsi générés dynamiquement à partir des données. Si
les données changent, il suffit de recompiler le document : tout se met
à jour automatiquement (textes, graphiques, analyses\ldots). Cela évite
les erreurs de copier-coller ou les résultats obsolètes.

\section{Mix entre texte et code intégré}\label{mix}

Lorsqu'on clique sur le bouton \textbf{Render} de RStudio (ou
combinaisons des touches Ctrl+Maj+K), un PDF est généré incluant à la
fois le texte et le code intégré. On peut intégrer du code comme suit en
l'encadrant entre une première ligne «
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r\}} » et
une dernière «
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}} » («
\texttt{\textasciigrave{}} » est le symbole accent grave, appelé aussi
\emph{backtick} ou \emph{backquote} depuis ses noms en anglais).

\begin{Shaded}
\begin{Highlighting}[]
\DecValTok{1} \SpecialCharTok{+} \DecValTok{1}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
[1] 2
\end{verbatim}

Ces sections de code intégré au texte et exécutable sont appelées des
\emph{chunks}. Quand on clique sur le bouton \textbf{+c} de RStudio (ou
Ctrl+Maj+I), un \emph{chunk} est automatiquement rajouté après
l'emplacement du curseur.

On peut aussi ajouter des options au code exécutable comme ceci
(derrière « \texttt{\#\textbar{}} », c'est-à-dire un croisillon et une
barre verticale) :

\begin{verbatim}
[1] 4
\end{verbatim}

L'option « \texttt{echo:\ false} » désactive l'impression du code (seul
le résultat final est affiché).

On peut arriver au même résultat avec la première ligne «
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r\}} »
remplacée par «
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r,\ echo=FALSE\}}
». Pas besoin alors de ligne avec l'option.

On peut également faire l'inverse, masquer le résultat et afficher le
code en remplaçant la première ligne «
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r\}} » par
«
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r,\ results="hide"\}}
».\\
Ou tout masquer en remplaçant la première ligne «
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r\}} » par
«
\texttt{\textasciigrave{}\textasciigrave{}\textasciigrave{}\{r,\ include=FALSE\}}
». Dans les deux derniers cas, les éléments du \emph{chunk} sont bien
exécutés comme dans les \emph{chunks} précédents où les résultats sont
affichés.

Exemple avec seulement le résultat :

\begin{verbatim}
[1] 7
\end{verbatim}

Exemple avec seulement le code :

\begin{Shaded}
\begin{Highlighting}[]
\DecValTok{3} \SpecialCharTok{{-}} \DecValTok{5}
\end{Highlighting}
\end{Shaded}

Exemple où tout est masqué :

L'intérêt de cacher tout un \emph{chunk} et son résultat est de gagner
de la place en n'affichant pas ce qui a peu de pertinence mais comme le
code est tout de même exécuté, on peut faire appel aux résultats
mémorisés dans ces \emph{chunks} ultérieurement, soit dans d'autres
\emph{chunks} (comme immédiatement ci-dessous) soit dans le texte comme
on le verra dans \hyperref[appelaunresultatr]{une section suivante}.

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{memoire }\SpecialCharTok{/} \DecValTok{8}
\end{Highlighting}
\end{Shaded}

\begin{verbatim}
[1] 1
\end{verbatim}

\subsection{Un autre exemple avec mix entre texte et code
intégré}\label{un-autre-exemple-avec-mix-entre-texte-et-code-intuxe9gruxe9}

L'exemple suivant est tiré du deuxième T.P. de « statistique avec R » du
cours de 1A de 2024-2025.

On charge des packages (\texttt{dplyr} et \texttt{ggplot2}).

Il est fréquent que les packages contiennent des bases de données. On va
appeler la liste des tables disponibles dans le package « dplyr » en
lançant la commande suivante :

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{data}\NormalTok{(}\AttributeTok{package=}\StringTok{"dplyr"}\NormalTok{)}
\end{Highlighting}
\end{Shaded}

Puis on va travailler sur la table « starwars ». Pour cela on charge la
table avec l'instruction suivante :

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{data}\NormalTok{(}\StringTok{"starwars"}\NormalTok{)}
\end{Highlighting}
\end{Shaded}

Cette table comporte notamment des informations sur les personnages de
la série de films \emph{Star Wars}. On veut maintenant étudier la
relation entre le poids (« mass ») et la taille (« height ») de ces
personnages. On commence par enlever de la table tous les personnages
avec au moins une valeur manquante pour ces deux variables :

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{starwars}\OtherTok{\textless{}{-}}\NormalTok{starwars }\SpecialCharTok{\%\textgreater{}\%} 
  \FunctionTok{filter}\NormalTok{(}\FunctionTok{is.na}\NormalTok{(height)}\SpecialCharTok{==}\ConstantTok{FALSE}\NormalTok{,}
         \FunctionTok{is.na}\NormalTok{(mass)}\SpecialCharTok{==}\ConstantTok{FALSE}\NormalTok{)}
\end{Highlighting}
\end{Shaded}

On représente la variable « mass » (ordonnées) en fonction de la
variable « height » (abscisses) sous forme d'un nuage de points :

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{ggplot}\NormalTok{(}\AttributeTok{data=}\NormalTok{starwars,}
       \FunctionTok{aes}\NormalTok{(}\AttributeTok{x =}\NormalTok{ height, }\AttributeTok{y=}\NormalTok{ mass ))}\SpecialCharTok{+}
  \FunctionTok{geom\_point}\NormalTok{()}
\end{Highlighting}
\end{Shaded}

}

\end{figure}%

On retire l'observation qui semble aberrante (au vu du graphique

» :

\begin{Shaded}
\begin{Highlighting}[]
\FunctionTok{ggplot}\NormalTok{(}\AttributeTok{data=}\NormalTok{starwars }\SpecialCharTok{\%\textgreater{}\%} 
         \FunctionTok{filter}\NormalTok{(mass}\SpecialCharTok{!=}\FunctionTok{max}\NormalTok{(mass)),}
       \FunctionTok{aes}\NormalTok{(}\AttributeTok{x =}\NormalTok{ height, }\AttributeTok{y=}\NormalTok{ mass ))}\SpecialCharTok{+}
  \FunctionTok{geom\_point}\NormalTok{()}
\end{Highlighting}
\end{Shaded}

Autre exemple où on « plante » la graine numéro \emph{n} de générateur

-- 31 nombres aléatoires selon la loi normale centrée réduite ;\\

Celle (\emph{idem}) du tirage des nombres entre 1 et 10 est :
\textbf{5.74}.

Pour avoir, en dehors d'un \emph{chunk}, une phrase reprenant un

Pour le dernier résultat : « \texttt{\textasciigrave{}} \texttt{r}

\subsection{Affichage de caractères
spéciaux}\label{affichage-de-caractuxe8res-spuxe9ciaux}

Points typographiques : pour que le symbole accent grave ne soit pas
compris par Quarto comme du code dans du texte, on doit l'entourer de
chaque côté par deux accents graves (cf.~le code de la section
précédente : si on est actuellement dans le mode \textbf{Visual}, pour
regarder ce code, cliquer sur le bouton \textbf{Source} de RStudio ou
Ctrl+Maj+F4).

Si la portion que l'on souhaite afficher comporte trois accents graves
(comme pour le début ou la fin d'un \emph{chunk}), on doit entourer
cette portion avec quatre accents graves de chaque côté pour les
désactiver (cf.~le début de \hyperref[mix]{la section « Mix entre texte
et code intégré »}).

Pour les autres caractères spéciaux (« * », « \textbackslash{} », « \#
», « \textless{} »\ldots{} liste non exhaustive), c'est la barre oblique
inversée (« \textbackslash{} »), appelée aussi \emph{backslash} ou
\emph{antislash}, qui sert à désactiver la fonction spéciale habituelle
de ces caractères en Quarto en la mettant juste devant chacun d'eux.

\subsection{Gras et italique}\label{gras-et-italique}

Les astérisques (« * ») servent à mettre en gras ou en italique du
texte. En encadrant avec un astérisque de chaque côté, on met *\emph{ce
texte en italique}* ; avec deux astérisques, on met **\textbf{ce texte
en gras}** ; avec trois, on le met à la fois ***\textbf{\emph{en gras et
en italique}}***. Pour ceux qui connaissent, cela utilise sur ce point
les mêmes conventions que la plateforme Discord.

\subsection{Couleur dans le texte}\label{couleur-dans-le-texte}

En dehors du code propre à R, pour mettre de la couleur dans le texte,
on doit au préalable mettre dans l'en-tête YAML «
\texttt{text:\ \textbar{}} » et, ligne suivante, «
\texttt{\textbackslash{}usepackage{[}dvipsnames{]}\{xcolor\}} ».

Ensuite, le texte à colorer (en rouge ici, « red ») doit être précédé
par « \texttt{\textbackslash{}textcolor\{red\}\{} » et suivi par «
\texttt{\}} ». Pour une autre couleur, on remplace « \texttt{red} » par
la couleur souhaitée en anglais.

Exemple : «
\texttt{\textbackslash{}textcolor\{ForestGreen\}\{Ce\ texte\ est\ vert\ foncé.\}}
» pour « \textcolor{ForestGreen}{Ce texte est vert foncé.} »

On peut choisir le noir, le blanc et 74 autres couleurs. Liste complète
des couleurs dans la section 4
d'\href{https://ctan.ceremade.dauphine.fr/macros/latex/contrib/xcolor/xcolor.pdf}{\emph{Extending
LaTeX's color facilities: the xcolor package}} (lien externe).

\subsection{\texorpdfstring{Parler « \texttt{code}
»}{Parler « code »}}\label{parler-code}

Lorsque l'on encadre une portion de texte avec un seul accent grave de
chaque côté, ce qui est encadré est affiché dans une police à chasse
fixe, c'est-à-dire une police où chaque caractère occupe même espace
horizontal que les autres caractères. C'est, dans les textes parlant
informatique, un type de police souvent utilisé pour représenter le
contenu du code utilisé. Exemples :

\begin{itemize}
\item
  si on tape « \texttt{\textasciigrave{}} \texttt{Machin}
  \texttt{\textasciigrave{}} » dans le code, cela affiche «
  \texttt{Machin} » où le M majuscule n'est pas plus large ou le i
  minuscule pas moins large que les autres lettres ;
\item
  alors que si on tape « \texttt{Machin} », ça affiche « Machin » dans
  la police choisie en début de document, idéalement dans une police où
  la chasse est différente pour chaque caractère.
\end{itemize}

\subsection{Principales différences entre typographies française et
anglaise}\label{principales-diffuxe9rences-entre-typographies-franuxe7aise-et-anglaise}

\subsubsection{Espace et ponctuation}\label{espace-et-ponctuation}

En français\footnote{Sauf en français du Canada.}, les signes de
ponctuation dits doubles car composés de deux éléments (« : », « ; », «
? » et «~!~») sont précédés par une espace. En anglais, pas d'espace
avant ces signes de ponctuation.

Pour les espaces avant les signes de ponctuation, le document Quarto sur
R gère cela au moment où le PDF est créé selon, si dans l'en-tête YAML,
on a indiqué « lang: fr » ou « lang: en ». Les espaces seront
normalement automatiquement rajoutées si le français est la langue
choisie ou supprimées pour l'anglais.

\subsubsection{Guillemets}\label{guillemets}

Le style de guillemets n'est pas le même. Guillemets français : « texte
» (avec espaces avant et après le texte\footnote{Sauf en français du
  Canada.}). Guillemets anglais des États-Unis : ``texte'' (sans
espaces). Guillemets anglais du Royaume-Uni~: `texte' (sans espaces).
Les guillemets anglais des États-Unis sont souvent utilisés en français
pour du texte entre guillemets imbriqués dans un extrait lui-même entre
guillemets. Exemple :

\begin{quote}
\emph{Il lui a alors répondu : « Mais vous m'aviez dit ``on y va !'' à
ce moment-là ».}
\end{quote}

Dans le PDF généré par un document Quarto sur R, le signe « \texttt{"} »
(dit guillemet droit ou guillemet informatique) est automatiquement
converti en guillemet anglais des États-Unis. Quarto arrive généralement
à reconnaître quel signe est le guillemet ouvrant et quel autre signe
est le guillemet fermant. De même le signe « \texttt{\textquotesingle{}}
» (dit apostrophe droite ou apostrophe informatique) est automatiquement
converti en guillemet anglais du Royaume-Uni ou en apostrophe normale («
\texttt{’} ») qui est une apostrophe plus soignée.

Pour une typographie irréprochable dans un rapport en français, il
faudra donc faire un copier-coller de guillemets français pour avoir les
bons guillemets.

\subsubsection{Espace insécable}\label{espace-insuxe9cable}

Attention, le PDF généré crée parfois des retours à la ligne après un
guillemet ouvrant ou avant un guillemet fermant. Une barre oblique
inversée (« \texttt{\textbackslash{}} ») peut alors être mis devant une
espace pour la transformer en espace insécable. Écrire donc
\texttt{«\textbackslash{}\ texte\textbackslash{}\ »} à la place de
\texttt{«\ texte\ »}. Rester attentif car l'espace insécable ne
fonctionne pas toujours si le texte commence ou termine par un caractère
spécial et non par une lettre ou un chiffre. Si le retour à la ligne se
fait encore à un endroit non désiré, voir si l'usage de l'italique ou du
gras n'est pas tout aussi pertinent que celui des guillemets ou si la
phrase ne peut pas être formulée un peu différemment.

Plus rarement, il peut y avoir des retours à la ligne non désirés en
d'autres endroits qu'entre guillemets. On peut utiliser la même méthode
avec la barre oblique inversée devant l'espace pour résoudre le
problème.

\subsubsection{Majuscules}\label{majuscules}

Pour les documents (articles, ouvrages, vidéos institutionnelles, etc.)
à teneur informative, technique ou scientifique, les titres en français
se limitent généralement à une majuscule au premier mot\footnote{Les
  œuvres à portée artistique ont des règles particulières pour les
  majuscules dans les titres mais seront rarement utiles dans un rapport
  universitaire. Si toutefois le besoin se fait sentir, on peut
  consulter les
  \href{https://fr.wikipedia.org/wiki/Wikip\%C3\%A9dia:Conventions_typographiques\#TITRES}{conventions
  typographiques de Wikipédia en français} qui compilent les normes les
  plus utilisées sur le sujet.} quand les titres en anglais ont une
majuscule à chaque mot important.

Les noms de langues ont une lettre initiale minuscule en français,
majuscule en anglais : « espagnol » et « arabe » mais « Spanish » et «
Arabic ». Même différence pour les adjectifs de nationalité ou pour les
noms de jours ou de mois (minuscule en français, majuscule en anglais).

\subsubsection{Abréviations}\label{abruxe9viations}

Les abréviations en français se terminent généralement (pas toujours)
par un point. Les abréviations anglaises du Royaume-Uni sont souvent
sans point quand les mêmes abréviations anglaises des États-Unis en
comportent souvent une. Ne pas hésiter à consulter des listes
d'abréviations courantes pour ne pas se tromper.

\subsection{Décimale et séparateurs de milliers}\label{virgule}

R utilise la typographie anglaise, et donc le point, pour le séparateur
entre partie entière et partie décimale.

Dans un rapport en français, pour le remplacer par une virgule comme en
français, écrire~ « \texttt{\textasciigrave{}} \texttt{r}
\texttt{format(round(mean(x\_tirage),\ 2),\ decimal.mark\ =\ ",")}
\texttt{\textasciigrave{}} »~ au lieu de\\
« \texttt{\textasciigrave{}} \texttt{r}
\texttt{round(mean(x\_tirage),\ 2)} \texttt{\textasciigrave{}} ».

Cela affichera « \textbf{5,74} » au lieu de « \textbf{5.74} ».

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{v }\OtherTok{\textless{}{-}} \DecValTok{1000} \SpecialCharTok{+} \DecValTok{5000}
\NormalTok{w }\OtherTok{\textless{}{-}} \DecValTok{10000} \SpecialCharTok{+} \DecValTok{51425}
\end{Highlighting}
\end{Shaded}

Ici on demande dans une phrase d'afficher les deux valeurs ci-dessus de
\emph{v} et de \emph{w} : « le premier résultat est 6000 et le second
est \ensuremath{6.1425\times 10^{4}} ». On le voit, par défaut quand
Quarto fait appel à des résultats de R, il n'utilise pas les séparateurs
de milliers (en effet, il écrit les nombres à 4 chiffres d'un seul bloc
et use d'une notation scientifique avec une puissance de 10 si les
nombres en comportent 5 ou plus).

Pour contraindre Quarto à générer le nombre sans notation scientifique
dans le PDF, on se sert d'une option supplémentaire dans la même
fonction \emph{format} déjà utilisé plus haut. Écrire alors «
\texttt{\textasciigrave{}} \texttt{r}
\texttt{format(w,\ scientific\ =\ FALSE)} \texttt{\textasciigrave{}} »
pour afficher « \textbf{61425} »

Problème qui apparaît, le séparateur de milliers n'est toujours pas
utilisé. Il faut donc le rajouter que le rapport soit en français ou en
anglais. Et rajouter une espace en français ou une virgule en anglais,
respectivement : « 61 425 » et « 61,425 ». Écrire alors dans un texte en
français « \texttt{\textasciigrave{}} \texttt{r}
\texttt{format(w,\ big.mark\ =\ "\ ",\ scientific\ =\ FALSE)}
\texttt{\textasciigrave{}} » pour afficher « \textbf{61 425} ».\\
Ou avec « \texttt{big.mark\ =\ ","} » si rapport en anglais.

\subsection{Écriture de formules mathématiques}\label{formule}

Les documents Quarto dans RStudio utilisent du langage LaTeX math pour
la rédaction de formules mathématiques. Une imbrication entre deux
symboles dollar (« \texttt{\$} »), un de chaque côté, sert à écrire des
formules en ligne au milieu du texte. Une imbrication entre quatre
symboles dollar, deux sur une première ligne et deux sur une dernière, à
les écrire dans un paragraphe centré par rapport aux marges gauche et
droite et propre à cette formule. Exemple :

\begin{quote}
Soient \(X_1, X_2, \ldots, X_n\) une suite de variables aléatoires
indépendantes et identiquement distribuées, telle que
\(\text{E}[X_i] = \mu\) et \(\text{Var}[X_i] = \sigma^2 < \infty\). On
définit :
\end{quote}

\begin{equation}\phantomsection\label{eq-sn}{
S_n = \frac{X_1 + X_2 + \cdots + X_n}{n}
      = \frac{1}{n}\sum_{i}^{n} X_i
}\end{equation}

Après chaque équation et les deux \texttt{\$\$}, on peut insérer entre
accolades un nom à l'équation. Ici elle a été appelée « sn », la
dernière ligne de mon code en LaTeX ci-dessus est «
\texttt{\$\$\ \{\#eq-sn\}} ». Chaque nom d'équation du rapport sera
précédé par « eq- ».

Pour appeler cette équation dans du texte, il suffit d'insérer dans
celui-ci « \texttt{@eq-sn} » pour donner «~Équation~\ref{eq-sn}~». Le
numéro de l'équation (créé par le document Quarto à la génération du
PDF) est alors cliquable dans ce PDF pour renvoyer vers l'équation en
question.

\section{Présentation des

\subsection{Passage à la ligne}\label{passage-uxe0-la-ligne}

Pour passer à la ligne sans créer un nouveau paragraphe dans le PDF, il
faut terminer une ligne par deux espaces et appuyer sur «
\texttt{Entrée} » afin d'écrire la ligne suivante. La barre oblique
inversée (« \textbackslash{} ») en fin de ligne a la même fonctionnalité
que les deux espaces.\\
Lorsque l'on passe à la ligne après deux espaces, il est possible lors
d'un enregistrement du document que RStudio transforme automatiquement
les deux espaces par une barre oblique inversée.

\subsection{Création d'un nouveau
paragraphe}\label{cruxe9ation-dun-nouveau-paragraphe}

Le plus simple (car le plus lisible) est de sauter une ligne en laissant
une ligne vide.

Si on passe à la ligne sans sauter de ligne et sans les deux espaces (ou
la barre oblique inversée) à la ligne précédente, cela écrira tout à la
suite dans le PDF.

De la même manière que décrit dans la section précédente, lorsque l'on
passe à la ligne après zéro ou une seule espace, il est possible lors
d'un enregistrement du document que RStudio supprime automatiquement le
passage à la ligne dans le code et ne garde qu'une espace.

\subsection{Retrait de paragraphe}\label{retrait-de-paragraphe}

Pour écrire tout un paragraphe avec un retrait à gauche plus important
(c'est-à-dire que la marge de gauche est provisoirement augmentée le
temps du paragraphe), il faut le commencer par un chevron pointant vers
la droite («~\texttt{\textgreater{}}~») et une espace. Si l'on souhaite
amplifier ce retrait, on cumule les chevrons («
\texttt{\textgreater{}\textgreater{}} »,
«~\texttt{\textgreater{}\textgreater{}\textgreater{}}~», etc.). Ce
retrait ne fonctionne que si ces caractères sont bien placés en début de
paragraphe, pas en début de ligne. Exemple :

\begin{quote}
\begin{quote}
Le théorème de Pythagore s'énonce fréquemment sous la forme suivante :
\end{quote}
\end{quote}

\begin{quote}
\begin{quote}
\begin{quote}
dans un triangle rectangle, le carré de la longueur de l'hypoténuse est
égal à la somme des carrés des longueurs des deux autres côtés.
\end{quote}
\end{quote}
\end{quote}

\begin{quote}
\begin{quote}
En nommant les sommets du triangle, le théorème peut se reformuler dans
l'implication suivante :
\end{quote}
\end{quote}

\begin{quote}
\begin{quote}
\begin{quote}
si un triangle \emph{ABC} est rectangle en \emph{C}, alors \emph{AB² =
AC² + BC²}.
\end{quote}
\end{quote}
\end{quote}

Plus haut, on a déjà vu les renvois du texte vers les
\hyperref[formule]{équations citées}, vers les
\hyperref[tableaux]{tableaux mentionnés} ou vers les
\hyperref[figures]{figures}.

\subsection{Notes de bas de page}\label{notes-de-bas-de-page}

Pour faire une note de bas de page, il faut mettre à l'endroit souhaité
un nom en chiffres ou en lettres précédé d'un accent circonflexe entre
crochets comme ceci : « \texttt{{[}\^{}8{]}} » et cela donne \footnote{Voici
  le contenu de la note de bas de page qui apparaîtra en bas de la même
  page.} puis dans un paragraphe suivant, on fait le rappel de la note
de la même façon avec le texte associé après deux-points.

Quel que soit le nom choisi pour le renvoi vers les notes de bas de
page, le document PDF généré numérote automatiquement les notes de page.
Et si on passe à la page suivante, la numérotation continue, on ne
recommence pas à 1.

À noter : lors d'un enregistrement, il se peut que RStudio ait
automatiquement remplacé les noms que vous avez choisi pour les notes de
bas de page par sa propre numérotation aussi dans le code (pas seulement
dans le PDF).

\subsection{Renvois vers une autre section du rapport}\label{renvois}

Voici un lien interne vers la \hyperref[pagedesreferences]{page des
références}. Le lien interne est écrit entre crochets tout de suite
suivi (sans espace) entre parenthèses d'un mot-clef (mot précédé d'un
croisillon) : «
\texttt{{[}page\ des\ références{]}(\#pagedesreferences)} ». Ce mot-clef
est répété à la section souhaitée, entre accolades et forcément derrière
le titre d'une section, en l'occurrence ici derrière le titre de la
section Références : « \texttt{\{\#pagedesreferences\}} ».

On peut aussi mettre ce lien interne dans une note de bas de page comme
ceci\footnote{Pour plus d'informations, se référer à la
  \hyperref[pagedesreferences]{page des références}.}.

Dans l'exemple ci-dessus, c'est un cas très utile de renvoi vers les
références. D'autres cas qui peuvent être très courants : renvoi vers
les annexes ou tout bonnement vers une section particulière du rapport
(pour comparer deux résultats notamment). Il faudra alors créer autant
de mot-clefs entre accolades tout au long du texte pour chaque endroit
vers lequel l'on devra pointer.

\subsection{Lien externe}\label{lienexterne}

Le lien externe fonctionne un peu de la même façon que le lien interne :
entre crochets tout de suite suivi (sans espace) entre parenthèses de
l'adresse de la page souhaitée.

Exemple : «
\texttt{{[}site\ officiel\ de\ l’Ensai{]}(https://www.ensai.fr/)} »
donne « \href{https://www.ensai.fr/}{site officiel de l'Ensai} ».

\section{Utilisation des
références}\label{utilisation-des-ruxe9fuxe9rences}

Dans l'en-tête YAML de ce document, on a «
\texttt{bibliography:\ references.bib} » qui va servir à faire une liste
pour la bibliographie. Pour faciliter son utilisation, cette liste va
être rédigée dans un fichier annexe appelé « references.bib ». Pour
créer ce fichier, aller dans le menu de R : \texttt{File} →
\texttt{New\ File} → \texttt{Text\ File} ; une fois ouvert, sauvegarder
le fichier sous le nom souhaité.

Pour l'exemple, voici un extrait le contenu du fichier « references.bib
» (extrait car la partie centrale remplacée par \texttt{{[}...{]}}) :

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{@article\{calhamer1974,}
\NormalTok{  author    = \{Allan B. Calhamer\},}
\NormalTok{  title     = \{The Invention of Diplomacy\},}
\NormalTok{  journal   = \{Games and Puzzles\},}
\NormalTok{  year      = \{1974\}, }
\NormalTok{  number    = \{21\}}
\NormalTok{\}}

\NormalTok{@book\{sharp1978,}
\NormalTok{  author    = \{Richard Sharp\},}
\NormalTok{  title     = \{The Game of Diplomacy\},}
\NormalTok{  year      = \{1978\},}
\NormalTok{  publisher = \{Arthur Baker\}}
\NormalTok{\}}

\NormalTok{[...]}

\NormalTok{@book\{kostick2015,}
\NormalTok{  author    = \{Conor Kostick\},}
\NormalTok{  title     = \{The Art of Correspondence in the Game of Diplomacy\},}
\NormalTok{  year      = \{2015\},}
\NormalTok{  publisher = \{Curses \& Magic\}}
\NormalTok{\}}

\NormalTok{@book\{vanmechelen2019,}
\NormalTok{  author    = \{Erik van Mechelen\},}
\NormalTok{  title     = \{How to Be a Diplomacy Player: A Philosophy of Face to Face Diplomacy\},}
\NormalTok{  year      = \{2019\},}
\NormalTok{  publisher = \{Lean Publishing\}}
\NormalTok{\}}
\end{Highlighting}
\end{Shaded}

On peut voir dans la \hyperref[pagedesreferences]{page des références},
la présentation finale. On avait choisi de lister les références dans
references.bib dans l'ordre chronologique ; attention, par défaut, cela
les affiche dans le PDF dans l'ordre alphabétique des auteurs, cela ne
respecte pas l'ordre initialement choisi (si on avait choisi de les
ranger d'une certaine façon).

Toujours dans l'en-tête YAML, « \texttt{nocite:\ "@*"} » permet de
lister dans le PDF la totalité des références mentionnées dans
references.bib. Sans cette ligne, seuls les éléments de la bibliographie
listés simultanément dans references.bib et cités dans le corps du
document seront listés dans le PDF.

Enfin, pour choisir où les lister, rajouter le code suivant à
l'emplacement voulu :

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{::: \{\#refs\}}
\NormalTok{:::}
\end{Highlighting}
\end{Shaded}

Pour citer un élément de la bibliographie dans le corps du document, il
faut faire précéder le mot d'appel à ce document précédé d'un arobase et
le tout entre crochets. Le mot d'appel est le terme composé des
caractères juste derrière l'accolade du document dans references.bib
donc ici « \texttt{calhamer1974} » pour l'article d'Allan B. Calhamer
publié en 1974, « \texttt{sharp19784} » pour le livre de Richard Sharp
édité en 1978, « \texttt{kostick2015} » pour celui de Conor Kostick en
2015 ou encore « \texttt{vanmechelen2019} » pour celui d'Erik van
Mechelen en 2019.

Cet appel dans le corps du texte sera cliquable et renverra à la ligne
de la publication dans le PDF si on choisit cette option dans l'en-tête
YAML : « \texttt{link-citations:\ true} ».

Exemples avec « \texttt{{[}@calhamer1974{]}} » et «
\texttt{{[}@kostick2015{]}} » :

Allan B. Calhamer, l'auteur américain du jeu de société
\emph{Diplomatie}, célèbre jeu basé sur les talents de stratège et de
négociateur des participants et conçu dans les années 1950, a écrit deux
décennies plus tard dans la presse un article très développé expliquant
le processus l'ayant amené à le créer
(\citeproc{ref-calhamer1974}{Calhamer 1974}).

Conor Kostick, historien irlandais et champion de ce jeu, a rédigé en
2015 \emph{The Art of Correspondence in the Game of Diplomacy}, un
ouvrage avec de nombreux conseils pour mieux gérer les phases de
négociation à \emph{Diplomatie} (\citeproc{ref-kostick2015}{Kostick
2015}).
